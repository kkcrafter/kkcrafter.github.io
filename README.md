# Kelvin Kwok — Portfolio Blog

A minimalist, multilingual portfolio blog built with Astro. Showcases work and writing on Sustainability/ESG, IT & Data, and Language Learning.

Live at: [kkcrafter.github.io](https://kkcrafter.github.io)

## Tech Stack

- **Framework:** [Astro](https://astro.build/) (SSG, zero client JS by default)
- **Styling:** Vanilla CSS with custom properties
- **Content:** Markdown with YAML frontmatter
- **Dev Environment:** Docker (Node.js 22 Alpine)
- **CI/CD:** GitHub Actions → GitHub Pages
- **i18n:** English, German, Chinese (`/en/`, `/de/`, `/zh/`)

## Quick Start

### Docker (recommended)

```bash
docker compose up --build
# → http://localhost:4321
```

### Native Node.js

```bash
npm install
npm run dev
# → http://localhost:4321
```

## Project Structure

```
src/
├── components/       # Astro components (Header, Footer, PostCard, etc.)
├── content/blog/     # Markdown posts organized by language
│   ├── en/           # English posts
│   ├── de/           # German posts
│   └── zh/           # Chinese posts
├── i18n/             # UI string translations + tag labels
├── layouts/          # Page layouts (Base, BlogPost, Landing)
├── pages/            # Route pages (auto-generated by Astro)
└── styles/           # Global CSS
```

## Content Workflow

### Adding a Post

1. Create a `.md` file in `src/content/blog/{lang}/` (e.g., `src/content/blog/en/my-post.md`)
2. Add frontmatter:

```yaml
---
title: "My Post Title"
description: "A short description."
pubDate: 2025-06-15
tags: ["it-dev", "docker"]
lang: "en"
draft: false
---
```

3. Write your content in Markdown below the frontmatter
4. Set `draft: true` to hide from production (visible in dev server)
5. Push to `main` to auto-deploy

### Translations

Use the **same filename** across language folders to link translations:

```
src/content/blog/en/docker-local-dev.md  ← English
src/content/blog/de/docker-local-dev.md  ← German translation
src/content/blog/zh/docker-local-dev.md  ← Chinese translation
```

Not every post needs all 3 languages. The language switcher automatically detects available translations.

### Tags

Tags are freeform strings. Add any tag you like — the blog listing page dynamically generates filter buttons from all existing tags. Suggested starters: `it-dev`, `esg`, `language-learning`, `learning-summaries`.

## Commands

| Command | Action |
|---------|--------|
| `npm run dev` | Start dev server at `localhost:4321` |
| `npm run build` | Build production site to `./dist/` |
| `npm run preview` | Preview production build locally |
| `docker compose up --build` | Start Docker dev environment |
| `docker compose down` | Stop Docker environment |

## Deployment

Push to `main` → GitHub Actions automatically builds and deploys to GitHub Pages.

The workflow is defined in `.github/workflows/deploy.yml`.
